Basic widgets

Widgets?

    View object interface for user interaction


    Android provides a set of fully implemented widgets:


  • Buttons
  • Checkboxes
  • Text entry fields
  • ... and many more!

    • You can also create your own by extending or combining existing widgets

Main widgets


  • TextView
  • EditText
  • ImageView
  • Button
  • RadioButton
  • ImageButton
  • CheckBox
  • ProgressBar

Where and how to declare widgets?

    Most common way - in xml

    <?xml version="1.0" encoding="utf-8"?>
  
<LinearLayout 
  xmlns:android="http://schemas.android.com/apk/res/android"
  android:layout_width="fill_parent" 
  android:layout_height="fill_parent" >
              
  <TextView android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:text="Hello, I am a TextView" />
            
  <Button android:layout_width="wrap_content"
          android:layout_height="wrap_content"
          android:text="Hello, I am a Button" />
            
</LinearLayout>

Where and how to declare widgets?

    Also possible in Java code

       @Override
   public void onCreate(Bundle savedInstanceState) {
       super.onCreate(savedInstanceState);
       
       TextView hello = new TextView(this);
       
       hello.setText("Hello, Android");
       
       setContentView(hello);
   }

Attributes

    Allow you to configure a widget's properties


    Defined inside a widget's tag as:

    android:attribute_name="attribute_value"

    where "android" is the namespace defined in the first tag of the file:

    <?xml version="1.0" encoding="utf-8"?>
<LinearLayout 
    xmlns:android="http://schemas.android.com/apk/res/android"
  
  ...

Attributes

    Some attributes are specific to a View object but inherited by its children

    Main attributes:

    android:id

android:layout_something

android:padding

android:text

    But there are many more - and you can create your own!

Input events

    User's interaction with your application

    Captured through Event Listeners:


  • onClick()
  • onLongClick()
  • onFocusChange()
  • onKey()
  • onTouch()
  • onCreateContextMenu()

How to catch input events - in Java

    Implement an interface of the input event type

    private final OnClickListener buttonListener = new OnClickListener() {
    @Override
    public void onClick(View v) {
      // do something when the button is clicked
    }
};

    Set that object in a view

    @Override
protected void onCreate(Bundle savedValues) {
    ...
    Button button = new Button();
    button.setOnClickListener(buttonListener);
    ...
}
