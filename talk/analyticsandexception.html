Analytics and exceptions

Analytics

  • It is important to know what are the user doing with your app
  • Track with analytics activities, buttons or any other event
  • Track silent exceptions
  • There are lots of libraries : Google Analytics, WebTrends, Flurry ...

Google analytics

  • Very stable implementation
  • It is the most used
  • Has an api that is too web-driven but is simple
  • The api is not polluted with to many methods like webtrends
  • Google analytics expose the data collected with apis
  • You can track installation with the market

Google analytics implementation

tracker = GoogleAnalyticsTracker.getInstance();
tracker.startNewSession(ANALYTICS_UID, 20, context);

tracker.stopSession();

//Tracking activities
tracker.trackPageView("/someActivity");

//Tracking events
tracker.trackEvent(BUTTON_CATEGORY, CLICK_ACTION, 
  label, position);

Exceptions

  • The Android Market is not a good tool to track exception as only a small percentage is reported
  • It is difficult to manage history on different version of the reported issue on the market
  • Use something else Bugsense or Acra

Acra

  • ACRA is a library enabling Android Application to automatically post their crash reports to a GoogleDoc form
  • Easy implementation
  • Very flexible but need a better user interface to handle the exceptions

Bugsense

  • ~acra
  • But very nice ui to handle the exceptions
  • You can redirect acra logs to bugsense
