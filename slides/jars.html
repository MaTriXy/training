Jars

General jar mechanics

  • Find a likely JAR
  • Put it in libs/
  • Code to the API, compile, use!

Unfortunately is not always so easy

Issues with Porting to Android - 1

  • Existing code might expect to be running on JavaSE, JavaME, JavaEE – not everything available
  • Presumed locations of files (read, write)
  • Windows-or-other tests fail
  • Wrappers around console applications
  • JNI
  • Older Java – need 1.5 or above

Issues with Porting to Android - 2

  • Java 7
  • Duplicate classes (example : org.json.*)
  • JAR dependencies
